# 소프트웨어 공학 및 개발 방법론

소프트웨어 공학은 **체계적이고 규율 있는 방법**으로 소프트웨어를 개발, 운영, 유지보수하는 공학적 접근입니다.

## 핵심 프레임워크

| 프레임워크 | 설명 |
|---|---|
| **Agile/Scrum** | 반복적·점진적 개발 방법론 |
| **DevOps** | 개발과 운영의 통합 문화 |
| **CMMI** | 소프트웨어 프로세스 성숙도 모델 |
| **TDD/BDD** | 테스트 주도 개발 방법론 |

## 학습 포인트

- 폭포수 vs 애자일 방법론 비교
- 마이크로서비스 아키텍처 설계
- CI/CD 파이프라인 구성
- 기술 부채 관리
